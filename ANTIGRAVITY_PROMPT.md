### 🧬 Dataset Schema

**Target:** `default` (0 = bad credit, 1 = good credit)

**Categorical features:** `checking_account`, `credit_history`, `purpose`, `savings_account`, `employment_since`, `personal_status`, `other_debtors`, `property`, `other_installments`, `housing`, `job`, `telephone`, `foreign_worker`

**Numerical features:** `duration_months`, `credit_amount`, `installment_rate`, `residence_since`, `age_years`, `existing_credits`, `num_dependents`

---

### ⚙️ Key Design Decisions

- **Preprocessing** (`utils/preprocessor.py`): LabelEncoder per categorical, StandardScaler for numericals, SMOTE for class imbalance (30/70 skew)
- **Model** (`models/train.py`): RandomForestClassifier (200 trees, balanced class weights, n_jobs=-1) is the default. GradientBoostingClassifier is the alternative.
- **Artifacts**: encoders + scaler saved as `.pkl` via joblib alongside the model in `models/`
- **API** (`api/app.py`): Flask, single prediction at `/predict`, batch at `/predict/batch`, gracefully handles unseen label values during inference
- **Risk levels**: `low` (≥0.70 prob_good), `medium` (0.40–0.70), `high` (<0.40)

---

### 📊 Current Model Performance

- ROC-AUC: ~0.79
- Accuracy: ~79%
- Good Credit F1: ~0.86
- Bad Credit F1: ~0.57 (imbalanced minority class)

### 🔒 Constraints & Preferences

- **Python 3.11+** only
- **No heredoc syntax** in terminal commands (zsh conflicts) — save scripts to `.py` files and run them
- Run **one terminal command at a time**
- **Never commit `.env`** or any secrets to git
- Keep preprocessing logic **centralized in `utils/preprocessor.py`** — don't duplicate it in train.py or app.py
- Imports: always add `sys.path.append(...)` at top of scripts that import from `utils/`
